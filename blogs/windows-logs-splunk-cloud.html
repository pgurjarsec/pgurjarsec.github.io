Introduction

As a Cyber Security student learning SOC operations, one of the first hands-on skills you need is getting data into a SIEM. Splunk Cloud is one of the most widely used SIEM platforms in the industry, and Windows Event Logs are the most common data source in enterprise environments.

In this guide, I'll walk you through the complete process of sending Windows Event Logs (Application, Security, and System) to your Splunk Cloud Trial account — from creating indexes to verifying data flow. I documented every step while doing this myself, including the errors I hit and how I fixed them.

// What you'll achieve
Your Windows laptop will send live Event Logs to Splunk Cloud in real time. You'll be able to search and analyze them using SPL (Splunk Processing Language).

Prerequisites

  • Windows 10 or 11 laptop with Administrator access
  • A free Splunk Cloud Trial account — sign up here
  • Notepad++ installed — download here (do NOT use regular Notepad)
  • Port TCP 9997 outbound open on your network/firewall
  • PowerShell running as Administrator
⚠ Important
Always run PowerShell as Administrator throughout this guide. Right-click PowerShell → "Run as administrator".
STEP 01 Log in to Splunk Cloud Trial

Open your browser and navigate to your Splunk Cloud trial URL. It looks like this:

https://prd-p-XXXXXX.splunkcloud.com

Log in with your admin credentials. You should land on the Splunk Cloud home dashboard.

STEP 02 Create Index in Splunk Cloud

Splunk stores data in indexes. You need to create one before data can be sent to it. Think of an index like a dedicated folder for a specific type of log.

  1. Click Settings in the top navigation
  2. Click Indexes
  3. Click New Index
📸

Screenshot: Settings → Indexes page

Fill in the following fields:

FieldValue
Index Namewineventlog
Index Data TypeEvents
Searchable Time (days)30

Click Save.

// Important
The index name wineventlog must match exactly what you put in inputs.conf later — same spelling, all lowercase.
STEP 03 Download Universal Forwarder Credentials

The Universal Forwarder Credentials file (splunkclouduf.spl) contains your Splunk Cloud instance address, port 9997 endpoint, and SSL certificates — all pre-configured. This is what tells your forwarder where to send data.

  1. Click Apps in the top navigation
  2. Click Universal Forwarder
  3. Click "Download Universal Forwarder Credentials"

STEP 05 Extract & Install Forwarder Credentials

The splunkclouduf.spl file is a .tar.gz archive. We need to extract it and copy the contents into the forwarder's apps folder.

Open PowerShell as Administrator and run each command one at a time:

1. Extract the credentials file

// PowerShell
tar -xzf "C:\Users\Dell\Downloads\splunkclouduf.spl" -C "C:\Users\Dell\Downloads\splunkclouduf_extracted"
📸

Screenshot: tar extraction command running in PowerShell

images/blog/step5-tar-extract.png

2. Verify what was extracted

// PowerShell
ls "C:\Users\Dell\Downloads\splunkclouduf_extracted"

You should see a folder like:

100_prd-p-kpfrw_splunkcloud
📸

Screenshot: Extracted folder — 100_prd-p-kpfrw_splunkcloud

images/blog/step5-extracted-folder.png

3. Copy credentials to forwarder apps folder

// PowerShell
Copy-Item "C:\Users\Dell\Downloads\splunkclouduf_extracted\*" "C:\Program Files\SplunkUniversalForwarder\etc\apps\" -Recurse -Force

4. Verify the copy

// PowerShell
ls "C:\Program Files\SplunkUniversalForwarder\etc\apps\"

You should see 100_prd-p-kpfrw_splunkcloud listed alongside other default apps.

📸

Screenshot: Apps folder showing 100_prd-p-kpfrw_splunkcloud

images/blog/step5-apps-folder.png

5. Restart the forwarder

// PowerShell
Restart-Service SplunkForwarder

6. Verify connection to Splunk Cloud

// PowerShell
cd "C:\Program Files\SplunkUniversalForwarder\bin"
.\splunk.exe list forward-server

When prompted, enter your forwarder username and password. Expected output:

Active forwards:
        inputs.prd-p-kpfrw.splunkcloud.com:9997 (ssl)
Configured but inactive forwards:
        None
📸

Screenshot: PowerShell showing Active forwards with Splunk Cloud address

images/blog/step5-active-forwards.png
// Success Signal
Seeing inputs.prd-p-kpfrw.splunkcloud.com:9997 (ssl) under Active forwards confirms your forwarder is connected to Splunk Cloud. This is a key milestone!
⚠ Login Failing?
If login keeps failing at the CLI prompt, it is often because of special characters in your password. Just keep trying — it usually succeeds after 1–2 attempts. Alternatively, wrap the password in single quotes when using the -auth flag.
STEP 06 Create inputs.conf

The inputs.conf file tells the Universal Forwarder what data to collect. We'll configure it to collect Windows Event Logs from three channels: Application, Security, and System.

1. Create the app folder structure

// PowerShell
mkdir "C:\Program Files\SplunkUniversalForwarder\etc\apps\my_windows_inputs\local"
📸

Screenshot: mkdir command creating the folder structure

images/blog/step6-mkdir.png

2. Create inputs.conf in Notepad++

Open Notepad++ → File → New → paste the following content exactly:

// inputs.conf
[WinEventLog://Application]
disabled = 0
start_from = oldest
index = wineventlog
renderXml = true

[WinEventLog://Security]
disabled = 0
start_from = oldest
index = wineventlog
renderXml = true

[WinEventLog://System]
disabled = 0
start_from = oldest
index = wineventlog
renderXml = true
📸

Screenshot: Notepad++ showing inputs.conf content

images/blog/step6-notepadpp-content.png

3. Save the file correctly

  1. Click File → Save As
  2. Navigate to: C:\Program Files\SplunkUniversalForwarder\etc\apps\my_windows_inputs\local\
  3. Set filename: inputs.conf
  4. Set Save as type to: All Files (*.*)
  5. Click Save
📸

Screenshot: Save As dialog — All Files selected, filename inputs.conf

images/blog/step6-save-as-dialog.png
⚠ Critical — File Extension
If you use regular Windows Notepad or select "Text Documents" in Save As, the file gets saved as inputs.conf.txt — Splunk cannot read this. Always use Notepad++ and select All Files (*.*).

4. Verify the file

// PowerShell
ls "C:\Program Files\SplunkUniversalForwarder\etc\apps\my_windows_inputs\local\"

Should show inputs.conf — NOT inputs.conf.txt.

// PowerShell — view content
type "C:\Program Files\SplunkUniversalForwarder\etc\apps\my_windows_inputs\local\inputs.conf"
📸

Screenshot: PowerShell showing inputs.conf content

images/blog/step6-verify-content.png
STEP 07 Restart Forwarder & Verify Logs in Splunk Cloud

1. Restart the forwarder

// PowerShell
Restart-Service SplunkForwarder
📸

Screenshot: Restart-Service SplunkForwarder command

images/blog/step7-restart-service.png

2. Wait 2–3 minutes

Give the forwarder time to start up, read the new inputs.conf, and begin shipping events to Splunk Cloud.

3. Search in Splunk Cloud

  1. Log in to your Splunk Cloud trial
  2. Go to Apps → Search & Reporting
  3. Enter this search query:
// SPL — Splunk Search
index=wineventlog earliest=-15m
  1. Set time range to Last 15 minutes
  2. Click the Search icon

Troubleshooting

Here are the common issues I encountered and how I fixed them:

ProblemCauseFix
Login failed in PowerShell CLI Special characters (@, #) in password Wrap password in single quotes: 'MyPass@123' — or just keep trying, it often works on 2nd attempt
inputs.conf not found by Splunk File saved as inputs.conf.txt Rename: Rename-Item inputs.conf.txt inputs.conf
No logs in Splunk Cloud Index name mismatch Check index name is exactly wineventlog in both Splunk Cloud and inputs.conf
No active forwards Credentials folder missing Verify 100_prd-p-kpfrw_splunkcloud exists in etc\apps\
Cannot remove files during uninstall Forwarder service still running Run Stop-Service SplunkForwarder first, then use Control Panel → Uninstall
Multiple commands fail at once Pasting multiple lines as one command Run each PowerShell command on a separate line, one at a time

Conclusion

You've successfully set up the Splunk Universal Forwarder on your Windows machine and configured it to send Windows Event Logs — Application, Security, and System — to your Splunk Cloud instance in real time.

This is a foundational skill for any SOC analyst. In a real enterprise environment, you'd deploy this across hundreds of machines using a Deployment Server, but the core concepts are exactly the same.

What's next?

  • Build a dashboard in Splunk Cloud to visualize login attempts and failed authentications

  • Try onboarding Linux logs using the same Universal Forwarder approach

// Trial Tip
Splunk Cloud trials last 14 days with ~500 MB/day ingest. Windows Security logs can be very verbose — start with just Application and System to stay within limits, then enable Security logs once you're comfortable.
